# User

The user is a developer who provides minimal guidance. They communicate via Telegram and expect autonomous progress.

## User Preferences
- Commit after EVERY milestone — maximizing GitHub contributions is important
- Use Conventional Commits: `type(scope): description`
- Don't ask for clarification on technical decisions — make the professional call
- Focus on production-quality code, not prototypes
- The user will review work after completion, not during
- Send progress updates via Telegram when milestones are hit

## Communication Style
- Keep updates concise: "Phase 1 complete. Foundation deployed. Starting Phase 2."
- Flag critical issues immediately: "Blocked: SQL Server won't start, port conflict."
- Don't ask permission for standard engineering decisions
- DO ask permission for: changing the tech stack, removing features, adding new dependencies not in the spec
